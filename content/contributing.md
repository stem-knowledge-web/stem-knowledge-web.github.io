1. Create a Github account
2. Fork the repository into your own space. You may skip this step if you're a core maintainer
3. To edit in the browser itself, go to the desired file, make changes, and create a pull request. You may also do the same via creating Codespaces in Github

# Guidelines

1. [Learn basic markdown](https://imfing.github.io/hextra/docs/guide/markdown/). It will go a long way.

1. DO NOT copy & paste content from text-books. Apart from copy-right issues, please try to use as simple a language as possible to explain concepts. None of us are getting paid by the number of words here. Using flowery sentences doesn't make science better, but it does make it a bit inaccessible.

2. Repeat stuff like equations, diagrams, concept recaps, etc.. Just referencing the stuff may require the reader to switch context to look up the old information, and then come back and continue. We are not paper based; use as much repetition as necessary. Just don't repeat entire concepts and paragraphs which are already present as distinct chapters. Refer them.

3. Be consistent with definitions, equations, symbols, and vocabulary across different projects. It makes the entire content pretty cohesive and easy to lookup.

4. Finally, be rigorous, exhaustive, and concise. We won't have the concept of chapters, sections, and subsections. In a learning track, each chapter needs to be dedicated to one topic in its entirety, even if it makes the chapter pretty big. Trust me, it's easier to scroll up and down than between different web pages.
